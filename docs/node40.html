Screen refresh events

For animations such as plots or games you set up a timer which then triggers a refresh request via “update()”. “Update” can be called as often as you like but QT will decide when to refresh the screen. 

void ScopeWindow::timerEvent( QTimerEvent *event )
          processData();
          calcScreenParameters();
          update(); // <----- triggers paint event
}

After “update()” has been called in the timer event then the paint event is executed by QT as soon as possible and re-paints the canvas of the widget: 

void ScopeWindow::paintEvent(QPaintEvent *) {
        QPainter paint( this );

        paint.drawLine( ... )
}
Note that neither the timer nor the “update()” function is called in a reliable way but whenever QT likes to do it. So the QT timers cannot be used to sample data but should only be used for screen refresh.
